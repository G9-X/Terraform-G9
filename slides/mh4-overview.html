MH4: API Gateway
trước Chat Lambda

Expose Chat Lambda qua API Gateway REST API với throttling và API Key authentication — biến raw Lambda thành production-grade API surface.

REST API Lambda Proxy Throttling API Key Auth
Problem → Solution

Vấn đề: Lambda endpoint không được bảo vệ

Trước (hiện tại)

  • API Gateway không yêu cầu auth
  • Không giới hạn request rate
  • Ai biết URL đều gọi được
  • Dễ bị abuse → Lambda cost tăng vọt
  • Bedrock Agent bị overload

Sau (MH4 hoàn thành)

  • API Key bắt buộc → 403 nếu thiếu
  • Throttling: 10 req/s, burst 20
  • Usage Plan quota: 1000 req/day
  • Lambda chỉ invoke khi auth pass
  • Cost predictable, DDoS-resistant

Implementation

Kế hoạch triển khai

Phase 1: Deploy stack (Terraform CI)

Push code → GitHub Actions tự động deploy toàn bộ infrastructure bao gồm API Gateway + Lambda. Stage prod sẵn sàng.

Phase 2: Thêm Usage Plan + API Key (Console)

Tạo Usage Plan (rate=10, burst=20, quota=1000/day), tạo API Key, associate với stage prod.

Phase 3: Bật API Key Required (Console)

Method Request → API Key Required = true trên POST /chat. Redeploy API.

Phase 4: Verify & Evidence

curl test 200 (có key) + 403 (không key). Screenshot cho Evidence Pack.
