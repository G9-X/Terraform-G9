02 / 07

Hiện trạng — Đã có gì?

Terraform module GeekBrain_Backend đã deploy sẵn REST API + Lambda Proxy. Chỉ cần thêm throttling + auth.

Client → API Gateway → POST /chat → Lambda (chat) → Bedrock Agent

✓ REST API Regional  |  ✓ Lambda Proxy Integration  |  ✓ CORS (OPTIONS)  |  ✓ Stage: prod
✗ Không throttling  |  ✗ Không authentication  |  ✗ Ai cũng gọi được

W5 yêu cầu bổ sung

① Throttling: Usage Plan với rate limit + burst limit
② Authentication: API Key required trên POST /chat
③ Test: curl 200 (có key) + curl 403 (không key)

03 / 07

Bước 1–2: Tạo Usage Plan & API Key

1

Tạo Usage Plan

Giới hạn số request/giây và burst để bảo vệ Lambda khỏi bị overload.

Console → API Gateway → Usage Plans → Create
2

Cấu hình Throttling

Rate: 10 requests/second  |  Burst: 20 requests
Quota: 1000 requests/day (tuỳ chọn cho demo)

Usage Plan → Throttling section → Set rate & burst
📸 Screenshot: Usage Plan creation form với Rate = 10, Burst = 20
3

Associate API Stage

Gắn Usage Plan vào stage prod của xbrain-chat-api.

Usage Plan → Associated Stages → Add Stage → xbrain-chat-api / prod
4

Tạo API Key

Tạo key mới, copy value, gắn vào Usage Plan.

Console → API Gateway → API Keys → Create API Key → Add to Usage Plan
04 / 07

Bước 3–4: Bật API Key Required & Re-deploy

5

Bật API Key Required trên POST /chat

Vào method POST → Method Request → API Key Required = true.

API Gateway → xbrain-chat-api → /chat → POST → Method Request → API Key Required ✓
6

Re-deploy API

Deploy lại API sang stage prod để thay đổi có hiệu lực.

API Gateway → xbrain-chat-api → Actions → Deploy API → Stage: prod
📸 Screenshot: Method Request với API Key Required = true

Lưu ý quan trọng

Sau khi đổi API Key Required, phải deploy lại mới có hiệu lực. Nếu quên deploy → test curl vẫn trả 200 dù không có key.

05 / 07

Bước 5: Test curl — 200 và 403

Test 1: Authenticated (200)

# Có API Key → 200 OK
curl -X POST \
  "https://{api-id}.execute-api.us-west-2.amazonaws.com/prod/chat" \
  -H "Content-Type: application/json" \
  -H "x-api-key: YOUR_API_KEY_HERE" \
  -d '{"question":"What is GeekBrain?"}'

# Response:
{"statusCode": 200, "body": {"answer": "..."}}

Test 2: Unauthenticated (403)

# Không có API Key → 403 Forbidden
curl -X POST \
  "https://{api-id}.execute-api.us-west-2.amazonaws.com/prod/chat" \
  -H "Content-Type: application/json" \
  -d '{"question":"What is GeekBrain?"}'

# Response:
{"message": "Forbidden"}

Kiến trúc sau khi hoàn thành MH4

Client → [API Key Header] → API Gateway → [Usage Plan: 10 req/s, burst 20]
                              ↓
                       POST /chat → Lambda Proxy → Bedrock Agent

✓ Throttling (10/s + burst 20)  |  ✓ API Key Authentication  |  ✓ CORS  |  ✓ Lambda Proxy

Bảo vệ

Không API Key → 403 ngay tại API Gateway, Lambda không bị invoke. Tiết kiệm cost + bảo vệ Bedrock Agent.

Throttling

Quá 10 req/s → 429 Too Many Requests. Burst cho phép spike tới 20 rồi steady ở 10. Ngăn DDoS amateur.
